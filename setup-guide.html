자동 리포팅 설정 가이드

1시간마다 Claude Code 사용량이 자동 전송됩니다. 5분이면 끝.

작동 방식

Claude Code 사용
→
로컬 JSONL 자동 기록
→
1시간마다 집계
→
챌린지 사이트
PC의 스케줄러가 로컬 파일에서 토큰 수(숫자)만 읽어 전송합니다. OAuth 토큰 불필요.

전송되는 데이터 (이게 전부)

🔢
토큰 수
📅
날짜
🏷
닉네임
🔄
세션 수
{ "nickname": "minjune", "date": "2026-04-08", "input_tokens": 1007, "output_tokens": 181207, "sessions": 3 }
대화 내용, 코드, 비밀번호, API 토큰 — 전송되지 않습니다.

설정 가이드

1
스크립트 다운로드
challenge-report.py 파일을 아래 경로에 저장하세요.
(챌린지 사이트 저장소에서 다운로드하거나, 관리자에게 받으세요)
# 저장 위치:
%USERPROFILE%\.claude\challenge-report.py

# 즉, 이런 경로가 됩니다:
C:\Users\내이름\.claude\challenge-report.py
2
최초 실행 — 닉네임 등록
명령 프롬프트(cmd)를 열고 아래를 입력하세요.
챌린지 사이트 닉네임과 비밀번호를 물어봅니다. 1번만 하면 됩니다.
python %USERPROFILE%\.claude\challenge-report.py
=== Claude Max 챌린지 — 초기 설정 ===
챌린지 닉네임: minjune
챌린지 비밀번호: ****
설정 저장 완료: C:\Users\...\challenge-config.json
[14:30] minjune | 2026-04-08 | 182,214 tokens | 3 sessions | OK
여기서 입력하는 닉네임/비밀번호는 챌린지 사이트 로그인 정보입니다.
Claude 계정 정보가 아닙니다. 로컬에만 저장됩니다.
3
1시간마다 자동 실행 등록
아래 명령어를 명령 프롬프트(cmd)에 한 번 입력하면,
PC가 켜져 있는 동안 매 시간 자동으로 사용량을 보고합니다.
schtasks /create /tn "ClaudeChallenge" /tr "python %USERPROFILE%\.claude\challenge-report.py" /sc hourly /st 00:00 /f
성공: 예약된 작업 "ClaudeChallenge"이(가) 만들어졌습니다.
✓
끝!
이제 Claude Code를 사용하면, 매 시간 사용량이 자동으로 챌린지 사이트에 반영됩니다.
아무것도 더 할 필요 없습니다.
확인 방법: 챌린지 사이트 → 인증 탭에서 "자동 리포팅 활성" 표시가 나오면 성공!
자동 실행 해제하려면:
schtasks /delete /tn "ClaudeChallenge" /f

challenge-report.py 파일을 아래 경로에 저장하세요.

~/.claude/challenge-report.py
2

1시간마다 자동 실행 등록
아래 명령어를 터미널에 한 번 입력하세요.
crontab에 매 시간 실행 스케줄이 추가됩니다.
# 기존 crontab 유지하면서 추가
(crontab -l 2>/dev/null; echo "0 * * * * python3 ~/.claude/challenge-report.py") | crontab -
# 확인하려면:
$ crontab -l
0 * * * * python3 ~/.claude/challenge-report.py
